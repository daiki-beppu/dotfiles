# コミットメッセージ規約

コミットメッセージは**日本語**で記述してください。

## 基本形式

```
<type>: <説明>

[任意: 本文]

[任意: フッター]
```

## タイプ

- **chore**: 依存関係のアップグレードなど、ルーティン/自動化されたタスク
- **deprecate**: 機能の非推奨化
- **feat**: 新機能の追加
- **fix**: バグやエラーの修正
- **release**: 新バージョンリリース関連

## ルール

- **説明**: 256文字以内、空にできません
- **本文・フッター**: 任意。記載する場合は前のセクションとの間に空白行を1行挿入し、各行は256文字以下にすること
- **破壊的変更**: 互換性を破る変更がある場合、タイプの後に ! を追加
  - 例: `fix!: 戻り値の型を変更`

## 例

```
feat: ユーザー認証機能を追加

ログイン、ログアウト、セッション管理機能を実装。
JWT トークンを使用した認証方式を採用。

関連: #123
```
