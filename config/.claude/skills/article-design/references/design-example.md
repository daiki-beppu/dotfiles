## 構成（遊び→実用グラデーション型）

### 1. 導入 — 両学長になりきってもらってみた！

- CLAUDE.md に「両学長として振る舞うこと」と書いて質問してみた会話例を掲載
- 「お金の大学」風の口調で返してくるインパクトを見せる
- 掴み：「CLAUDE.md に1行書くだけで、Claude Code の"人格"が変わる」
- **会話例は執筆時に生成する**

### 2. 基本 — CLAUDE.md に書くだけでロール設定できる

- 設定方法の解説（CLAUDE.md の `## 振る舞い` セクションに書く）
- **同じ質問を3キャラに投げた会話例を並べて比較**
  - かわいい妹キャラ
  - ツンデレイケメン
  - 厳しい上司
- 「プロンプトを変えるだけで返答のトーンがガラッと変わる面白さ」を体感させる
- **会話例は執筆時に生成する**

### 3. 実践 — ロール設定のメリットを実例で紹介

- 厳しいレビュアーにコードを添削してもらう
- 優しいメンターにエラーの相談をする
- 専門家（セキュリティエンジニア等）になりきってアドバイスをもらう
- 「遊び」だけでなく「実務で使える」ことを示す転換点

### 4. 応用 — キャリアメンターを作ってガチの転職壁打ちをした話（実体験）

- **CLAUDE.md でのロール定義**
  - 「優秀なエンジニアのキャリアメンターとして振る舞うこと」の実コード
- **Knowledge Base の構造化**
  - 経歴・スキル・転職軸を Knowledge Map として整理した話
  - `career/profile.md` の Who / What / Why / Where 構造
- **ペルソナファイルで専門家を使い分け**
  - キャリアアドバイザー（市場データ重視、客観的）
  - シニアエンジニア（経験ベース、厳しめ）
  - 同じテーマでもペルソナで対話の質が変わる実例
- **カスタムスキルのルーティング**
  - 相談内容に応じて自動的に最適なサブスキルに切り替え
  - 自己分析 → 企業分析 → 面接対策 → 条件交渉まで壁打ちした実体験
- **実際に転職活動の意思決定に活用している実例**

### 5. まとめ

- ロール設定は遊びから実用まで幅広い
- 1行から始められる → 深めたければスキル + ペルソナ + Knowledge Base
- 「AI の可能性が広がる」
