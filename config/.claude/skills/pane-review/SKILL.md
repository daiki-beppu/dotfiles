# pane-review

隣の cmux ペインの内容を読み取り、指定ペルソナの視点でレビューし、結果を隣のペインに送信する。

並列 Claude セッション（parallel スキルなど）で作業中、別ペインの plan やコードを
第三者の専門家視点でレビューしたい場面で使う。自分でレビューするより、
専門家のレンズを通すことで見落としがちな観点（テスト戦略、設計原則など）を拾える。

## 引数

- 第1引数: ペルソナ名（省略時: `t-wada`）
- 例: `/pane-review`, `/pane-review t-wada`, `/pane-review sandi-metz`

## 実行フロー

### 1. ペルソナ読み込み

引数からペルソナ名を取得（デフォルト: `t-wada`）し、
`skills/pane-review/references/personas/<name>.md` を Read ツールで読む。

ペルソナファイルには「レビュー観点」と「トーン」が定義されており、
これがレビューの品質と一貫性を担保する。
ファイルが見つからなければ「ペルソナ `<name>` が見つかりません」とエラーを出して終了。

### 2. 対象ペイン特定

```bash
cmux list-panes
```

`$CMUX_SURFACE_ID` で自分のペインを識別し、それ以外のペインを対象とする。
自分のペインを除外することで、誤って自分自身をレビューすることを防ぐ。

- 対象が1つ → そのペインを使う
- 対象が0 → 「他のペインが見つかりません」とエラー
- 対象が複数 → 一覧を表示し、AskUserQuestion でどれをレビューするか確認

### 3. 画面読み取り

```bash
cmux read-screen --surface <target_surface_id> --scrollback
```

スクロールバックを含めて取得する。計画やコードは画面外にスクロールしていることが多いため、
`--scrollback` がないと不完全な情報でレビューしてしまう。

### 4. レビュー実行

ペルソナファイルの「レビュー観点」に沿って、取得した内容を一つずつ評価する。
「トーン」に従うことで、建設的で受け入れやすいフィードバックになる。

レビュー結果のフォーマット:
```
## [ペルソナ名] によるレビュー

### 総評
（全体的な印象を1-2文で）

### 指摘事項
1. （観点ごとの具体的な指摘と改善提案）
2. ...

### 良い点
- （良い点があれば言及）
```

### 5. 結果送信

レビュー結果を隣のペインに送信する。

`cmux send` はターミナル入力のエミュレーションなので、長文を直接送ると
改行やエスケープで壊れる。一時ファイル経由で渡すのが安全：

1. レビュー結果を `/tmp/pane-review-result.md` に書き出す
2. 隣のペインが Claude セッションの場合:
   - ファイルパスを伝えて読ませる: `cmux send --surface <id> "/tmp/pane-review-result.md を読んでレビューフィードバックを反映して\n"`
3. 隣のペインが Claude セッションでない場合:
   - `cat` コマンドで表示: `cmux send --surface <id> "cat /tmp/pane-review-result.md\n"`

### 6. 完了報告

自分のペインに「レビューを送信しました」と表示して終了。

## 前提条件

- cmux が利用可能であること（`$CMUX_SURFACE_ID` が設定されていること）
- ペインが2つ以上存在すること

## ペルソナの追加方法

`references/personas/<name>.md` にファイルを追加する。必要な項目:
- 人物の背景（1-2行）— どんな専門家か
- レビュー観点（箇条書き）— 何をチェックするか
- トーン（レビューの口調・姿勢）— どう伝えるか
