# lifeplan-input

リベシティのライフプランツール（lifeplan.libecity.com）に収支データを自動入力する。

## 前提条件

- Chrome DevTools MCP が利用可能であること
- 1Password CLI (`op`) が利用可能で、`Personal` vault に `リベシティ` アイテムがあること
- 入力する収支データが用意されていること（mf-scrape の出力 or ユーザー提供）

## 入力データ形式

mf-scrape の出力と同じ JSON 形式を期待する：

```json
{
  "period": "2026/03/01 - 2026/03/31",
  "income": 221316,
  "expense": 254667,
  "expense_breakdown": [
    { "category": "食費", "total": 16014, "subcategories": [...] },
    { "category": "日用品", "total": 13309, "subcategories": [...] }
  ]
}
```

## カテゴリマッピング（固定）

MF のカテゴリからライフプランツールの入力項目へのマッピング：

| MF カテゴリ | ライフプラン項目 | ステップ |
|---|---|---|
| 当月収入 | 給与 | 2/7 収入 |
| 住宅 + ローン返済 | 住宅 | 3/7 固定費 |
| 通信費(情報サービス) + 通信費(住宅内) | 通信費 | 3/7 固定費 |
| サブスクリプション | サブスク費 | 3/7 固定費 |
| 食費合計 | 食料品 | 4/7 変動費 |
| 日用品合計 | 日用品 | 4/7 変動費 |
| 衣服・美容合計 | 衣服・美容 | 4/7 変動費 |
| 交通費合計 | 交通費 | 4/7 変動費 |
| 社会保険料 | 税金 | 5/7 不定期固定費 |

マッピングに該当しないカテゴリ（趣味・娯楽、自動車、現金・カードなど）はスキップする。未分類が大きい場合はユーザーに確認を促す。

## 実行フロー

### 1. ログイン情報の取得

1Password から直接取得する：

```bash
LIFEPLAN_EMAIL=$(op read "op://Personal/リベシティ/username")
LIFEPLAN_PASSWORD=$(op read "op://Personal/リベシティ/password")
```

### 2. ログイン

Chrome DevTools MCP でライフプランツールにアクセス：

```
new_page → https://lifeplan.libecity.com/
```

ログインページにリダイレクトされた場合：

1. `take_snapshot` でフォーム構造を確認
2. メールアドレス入力フィールドに `LIFEPLAN_EMAIL` を `fill`
3. パスワード入力フィールドに `LIFEPLAN_PASSWORD` を `fill`
4. 「ログイン」ボタンをクリック
5. ホーム画面が表示されるのを `wait_for` で待つ

### 3. 入力開始

ホーム画面の「入力を再開する」リンクをクリック：

```
click → "入力を再開する" (url: /actual/input)
```

### 4. Step 1/7: 期間選択 (step=period)

データの期間に応じて適切な期間ボタンをクリック：
- 1年分のデータ → 「1年」
- 半年分 → 「半年」
- 3ヶ月分 → 「3ヶ月」
- それ以外 → 「カスタム」

### 5. Step 2/7: 収入 (step=income)

入力データの `income` を「給与」に入力する。

各項目の入力パターン（全ステップ共通）：

1. 項目ボタンをクリック → 編集ダイアログが開く
2. `take_snapshot` で金額入力フィールドの uid を確認
3. 金額フィールドをクリックして値を入力（`type_text`）
4. 「更新」ボタンをクリック

入力モードの違い：
- **「毎月同じ」**: 月額をそのまま入力
- **「期間の合計」**: 月額 × 月数 を入力（平均が月額になる）

「次へ」をクリックして次のステップへ。

### 6. Step 3/7: 毎月の固定費 (step=enemy1)

マッピングに従って以下を入力：

| 項目 | 値の算出方法 |
|---|---|
| 住宅 | MF「住宅」サブカテゴリの「住宅」+ 「ローン返済」 |
| 通信費 | MF「通信費」合計 + MF「住宅」内の「通信費」 |
| サブスク費 | MF「住宅」内の「サブスクリプション」 |

水道光熱費、自己投資(リベシティ) は MF データに該当がなければ 0 のままスキップ。

### 7. Step 4/7: 毎月の変動費 (step=enemy2)

| 項目 | 値の算出方法 |
|---|---|
| 食料品 | MF「食費」合計 |
| 日用品 | MF「日用品」合計 |
| 衣服・美容 | MF「衣服・美容」合計 |
| 交通費 | MF「交通費」合計 |

交際費は MF データに該当がなければ 0 のままスキップ。

### 8. Step 5/7: 不定期の固定費 (step=enemy3)

| 項目 | 値の算出方法 |
|---|---|
| 税金 | MF「住宅」内の「社会保険料」 |

### 9. Step 6/7: 不定期の変動費 (step=enemy4)

MF データに「特別費」「旅行」に該当するカテゴリがあれば入力。なければスキップ。

### 10. Step 7/7: 投資 (step=investment)

MF データに投資関連のカテゴリがあれば入力。なければスキップ。

### 11. 完了確認

入力サマリー画面のスクリーンショットを撮影してユーザーに確認を促す。問題なければ「実績入力を完了」ボタンをクリック。

## 引数

- `/lifeplan-input` : mf-scrape の直前の出力データを使用
- データを直接指定する場合はチャットで JSON を渡す

## 注意事項

- ライフプランツールの「期間の合計」モードでは、月額 × 期間月数 の値を入力する必要がある（自動計算される平均が月額になるように）
- 入力済みのデータがある場合は上書きされるため、ユーザーに確認を取ること
- ダイアログの入力フィールドは毎回 `take_snapshot` で uid を確認すること（動的に変わる）
