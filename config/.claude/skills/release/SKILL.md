# release — GitHub Release & npm publish パイプライン

## Overview

前回リリースからの差分を分析し、バージョン自動判定 → リリースノート生成 → ユーザー確認 → リリース作成までを一気通貫で実行する。

## When to Use

- `main` ブランチ上で新しいバージョンをリリースしたいとき
- `/release` コマンドを実行したとき
- 「リリースして」「リリース作って」「npm に公開して」と言われたとき

## パイプライン

### Step 0: Context 収集

```bash
git branch --show-current                    # 現在のブランチ
gh release list --limit 1                    # 最新リリース
git log <last-tag>..HEAD --oneline           # 前回リリース以降のコミット
gh pr list --state merged --base main --json number,title,mergedAt  # マージ済み PR
```

**ゲート**: `main` ブランチ以外にいる場合はここで停止し、main に切り替えるか確認する。

**ゲート**: 前回リリース以降のコミットがない場合は「リリースする変更がありません」と表示して終了する。

### Step 1: バージョン自動判定

前回リリース以降のコミットメッセージを分析し、セマンティックバージョニングに基づいてバージョンを決定する。

| コミットタイプ | バージョン変更 |
|---------------|--------------|
| `!` 付き（破壊的変更） | **major** バンプ |
| `feat` あり | **minor** バンプ |
| `fix` / `chore` のみ | **patch** バンプ |

判定結果をユーザーに表示する（例: `v0.3.0 → v0.4.0 (minor: feat コミットあり)`）。
ユーザーが別のバージョンを指定した場合はそちらを採用する。

### Step 2: リリースノート生成 & 確認

**PR 単位で差分を分析**し、ユーザー目線のリリースノートを生成する。
PR の詳細は `gh pr view <number>` で取得する。

**カテゴリ**（該当があるもののみ表示）:

- **Features** — 新機能・機能追加
- **Bug Fixes** — ユーザーに影響するバグ修正
- **Security** — セキュリティ関連の改善
- **Infrastructure** — テスト基盤・CI/CD 等の改善

**リリースノートのルール**:

- 内部実装の詳細（リファクタ、テスト修正、lint 修正等）は含めない
- ユーザーにとっての価値が伝わる表現にする
- 各項目に PR 番号を付与する

生成したリリースノートをユーザーに表示し、承認を得る。修正指示があれば反映する。

### Step 3: バージョン更新 & プッシュ

1. `package.json` の `version` フィールドを更新する
2. `/skills commit-convention` に従いコミットする（タイプ: `release`）
3. `git push` で main にプッシュする

### Step 4: GitHub Release 作成

```bash
gh release create v<version> --target main --title "v<version>" --notes "<リリースノート>"
```

作成後、リリース URL を表示する。
publish ワークフローが設定されている場合は、自動で npm publish が実行される旨を伝える。

## エラーハンドリング

| 状況 | 対応 |
|------|------|
| main 以外のブランチ | 停止し、main への切り替えを確認 |
| 前回リリースからの差分なし | 「リリースする変更がありません」と表示して終了 |
| `gh` 認証エラー | `gh auth login` を案内 |
| 同じタグが既に存在 | エラー表示し、バージョンの再指定を促す |

## Rules

- 各ステップは順番に実行する
- リリースノートは必ずユーザーの承認を得てからリリースを作成する
- コミットメッセージは commit-convention に従う
- バージョン判定はユーザーが上書き可能
