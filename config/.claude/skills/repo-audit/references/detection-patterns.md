# repo-audit 検出パターン集

4 カテゴリそれぞれの「ヒットさせたいキーワード」「前後確認のポイント」「誤検出を避けるための除外文脈」をまとめる。SKILL.md 本文は判断フローに専念するため、具体パターンはこのファイルを出発点にする。

> ⚠ **自スキル除外**: このファイル自身および `SKILL.md` に記載されたキーワード列（`console.log` / `TODO` / `catch {}` / `deprecated` など）は、repo-audit の全カテゴリで検出対象から除外する（SKILL.md の Rules 参照）。

---

## カテゴリ 1: 過剰な防御 / 抽象化 / 半端な実装

| 種別 | 検出の目印 | 前後確認のポイント |
|------|-----------|------------------|
| デバッグ出力残り | `console.log`, `print(`, `dbg!`, `debugger;` | 本番コードか、開発補助か。CLI の意図的な stdout 出力は除外 |
| 到達しないコメント | `TODO`, `FIXME`, `XXX`, `HACK` | 意図的マーカー（`TODO(human)` 等）は除外 |
| 空の防御 | `catch {}`, `except Exception: pass` | コメントで理由が書かれているか |
| 未完成実装 | `throw new Error("not implemented")`, `pass  # TODO`, `return null  // later` | テスト用スタブか本番コードか |
| 無意味な抽象化 | 1 箇所でしか呼ばれないラッパー関数、元関数を素通し | `git grep` で呼び出し元カウント |

**ルール**: Grep ヒットだけで issue 化せず、**必ず前後を Read** で確認してから判定する。

---

## カテゴリ 2: ドキュメントの陳腐化

| ドキュメント | 突合対象 | 典型的な陳腐化パターン |
|-------------|---------|---------------------|
| `README.md` | `scripts/`, `Makefile`, `package.json`, `flake.nix` | 「`nr build` でビルド」と書いてあるがスクリプト側に build が無い / セットアップ手順が旧バージョンのまま |
| `CLAUDE.md`（プロジェクト直下） | 実ディレクトリ構成、運用ルール、口調指定 | 「config/ 配下に配置」が実態と違う / 削除済みコマンドへの言及が残る |
| 各 `SKILL.md` | 自スキルの description と本文、他スキルへの参照 | description で宣伝する機能が本文に無い / 例示コマンドの引数が古い |
| `config/.claude/CLAUDE.md` | 他の CLAUDE.md・各スキルとの整合 | グローバル規約と個別スキルが矛盾 |

**重大度の目安**:

- **high**: 参照先（コマンド・ファイル・スキル名）が完全に存在しない
- **medium**: 存在はするが引数や使い方の記述が旧式
- **low**: 表現の古さ・誤字・口調の不一致

**行単位に落とせない "方針ズレ"** は該当見出し単位で 1 件として起票する（行番号は見出し位置）。

---

## カテゴリ 3: 実装とスキルの乖離

各 SKILL.md が参照する以下が実在するかを検証する:

- **外部コマンド**（`gh`, `nr`, `nlx`, `darwin-rebuild` 等）: `command -v <cmd>` で PATH 確認
- **ファイルパス**（`config/.claude/settings.json` 等）: リポジトリ内に存在するか
- **他スキル参照**（`/skills simplify`, `Skill ツールで xxx を呼び出す`）: `ls config/.claude/skills/` と照合
- **settings.json の hooks / permissions**: 参照されるコマンドが実行可能か

逐次 Bash で確認できる軽量カテゴリなので、Grep → Read → 実在チェックの 3 段で完結する。

---

## カテゴリ 4: 意味のない後方互換維持

| 種別 | 検出の目印 |
|------|-----------|
| 明示マーカー | `deprecated`, `legacy`, `旧`, `old-`, `-v1`, `_old`, `backward-compat` |
| 参照切れの移行告知 | 「以前は X、現在は Y」という記述で、X が他から参照されていない |
| 未使用エイリアス / ラッパー | 関数・コマンド・`zsh abbr`・skill 名の旧名定義で、呼び出し 0 件 |
| 廃止フラグの残存 | `settings.json` / `flake.nix` 等で、現行ツールが受け付けないオプションや削除済み hook / permission 参照 |
| 到達不能な互換分岐 | 「バージョン X 未満なら〜」の条件分岐で、条件を満たす実行環境が存在しない |

**確認手順**（Grep だけで判定しない）:

1. 上記キーワードを Grep で抽出
2. Read で前後を読み、意図（現役の互換維持 vs 残骸）を判定
3. エイリアス・関数・skill 名は `git grep -n <旧名>` で全リポジトリ参照を集計。定義箇所以外 0 件なら「未使用」
4. `git log -S <旧名>` で、一度削除された参照が復活していない履歴を確認
5. 現行ツールの受理オプションとの突合（例: `darwin-rebuild --help`, `claude --help`）

**重大度の目安**:

- **high**: 参照先が完全に存在しない / 分岐が到達不能
- **medium**: 参照はあるが実質呼び出し 0
- **low**: コメントやドキュメント記述だけ残っている

**除外対象**: 現在も意図的に維持している互換（移行期間中のエイリアス、他リポジトリ向けマイグレーションガイド等）は検出から外す。
