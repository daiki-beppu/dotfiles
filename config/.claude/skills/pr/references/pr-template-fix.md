# PR テンプレート: fix（バグ修正）

fix/ ブランチで PR を作成する際に使用するテンプレート。
バグの再現性と原因の特定に重点を置く。

## テンプレート

```
## 概要
<!-- 何を修正したかを1文で -->
Fixes #番号

## バグ詳細
- **症状**: <!-- ユーザーから見える問題 -->
- **再現条件**: <!-- どういう状況で発生するか -->
- **再現方法**: <!-- 具体的な手順 -->

## 原因
<!-- 問題の根本原因を説明 -->

## 修正内容
<!-- 何をどう変更したか -->
-

## 影響範囲
<!-- この修正が影響する機能・コンポーネント -->

## テスト計画
<!-- バグが再現しないことの確認手順 -->
- [ ]
```

## 記述ガイド

### バグ詳細
レビュアーがバグを理解するために必要な情報をまとめる。
再現方法は具体的なステップで書き、誰でも再現できるようにする。

### 原因
コードレベルで何が問題だったかを説明する。
「どのコードの」「どういうロジックが」「なぜ問題を引き起こしたか」を含める。

### 影響範囲
修正による副作用がないことを確認するために、
変更が影響しうる機能やコンポーネントを列挙する。

### テスト計画
最低限、以下を含める:
- 報告されたバグが再現しないことの確認
- 関連する正常系が壊れていないことの確認
