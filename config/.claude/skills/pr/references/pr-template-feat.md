# PR テンプレート: feat（新機能）

feat/ ブランチで PR を作成する際に使用するテンプレート。
設計判断の記録と背景の共有に重点を置く。

## テンプレート

```
## 概要
<!-- 何を追加したか、なぜ必要かを1-3文で -->
Closes #番号

## 設計判断
<!-- 採用したアプローチと、比較検討した代替案 -->
<!-- なぜこの方法を選んだかの理由を含める -->
-

## 変更内容
<!-- 主な変更点を箇条書きで -->
-

## テスト計画
<!-- セルフ e2e の手順 -->
- [ ]
```

## 記述ガイド

### 概要
変更の目的と背景を簡潔に。「何を」「なぜ」が伝わることが重要。
Closing keyword（`Closes #番号`）は概要の末尾に置く。

### 設計判断
将来の自分やレビュアーが「なぜこう実装したのか」を理解できるように書く。
トレードオフがあった場合は、選ばなかった選択肢とその理由も記述する。

例:
- JWT を採用（セッション方式と比較して、ステートレスでスケーラブル）
- ミドルウェアパターンで実装（各ルートで個別に認証するより保守性が高い）

### テスト計画
セルフ e2e テストの手順をチェックリストで記述する。
実際に手動で確認する操作を、再現可能な形で書く。
