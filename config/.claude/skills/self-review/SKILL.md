# self-review — 品質チェックパイプライン

## Overview

変更コードに対して品質チェックを順番に実行するオーケストレータースキル。
各ステップは既存スキルまたはインライン実装で行い、問題があれば自動修正して報告する。
コミットやプッシュは行わず、修正はワーキングツリーに残す。

**承認待ちで停止しない**: 指摘の扱いは本スキル内で閉じ、対話的なコマンドは呼び出さない。

## When to Use

- コードの品質チェックだけ実行したいとき
- WIP の状態で問題がないか確認したいとき
- PR を出す前のセルフレビューとして
- `/self-review` コマンドを実行したとき
- 「セルフレビューして」「品質チェック」「WIP チェック」と言われたとき

## パイプライン

### Step 0: Context 収集

以下のコマンドで現在の状態を確認する。依存がないため 1 ターンで並列発行する：

```bash
git branch --show-current           # 現在のブランチ
git status --short                  # ステータス
git diff --stat main..HEAD          # 変更ファイルの概要
git log main..HEAD --oneline        # main からの差分コミット
```

**ゲート**: 現在のブランチが `main` または `master` の場合はここで停止し、feature ブランチの作成を促す。

### Step 1: コード簡素化レビュー

`simplify` スキルを Skill ツールで呼び出す（`Skill(skill="simplify")`）。

変更されたコードの品質・可読性・再利用性をレビューし、問題があれば自動修正する。
修正内容をユーザーに報告して次のステップへ進む。

### Step 2: セキュリティ点検（インライン）

built-in の `/security-review` は CRITICAL 指摘で対話的に停止するため、**呼ばない**。
代わりに差分をインラインで点検する。ここは止まらないことが最優先。

**手順**:

1. `git diff main..HEAD -- ':!*.lock' ':!*.snap'` の **追加行（`^+`）** を対象にする
2. 以下の観点で Grep / 目視チェック（パターンは ripgrep に渡す前提）:
   - **シークレット**: `AKIA[0-9A-Z]{16}`（AWS アクセスキー）、`xox[baprs]-`（Slack トークン）、`-----BEGIN .* PRIVATE KEY-----`、`password\s*=\s*["'][^"']+["']`、`token\s*=\s*["'][^"']+["']`、`api[_-]?key\s*=\s*["'][^"']+["']`
   - **インジェクション**: 文字列連結の SQL、`exec(` / `eval(` の動的呼び出し、未サニタイズ入力を含むシェル `$(...)`
   - **意図せぬ混入**: `.env` / `credentials*` / `*.pem` / `id_rsa*` のステージング
3. 分類して扱う:
   - **CRITICAL**（実シークレット / 明らかな脆弱性 / 秘密ファイルの混入）: 即座に自動修正する。secret はプレースホルダ化（例: `sk-real-xxx` → `<REDACTED>`）、秘密ファイルは `git restore --staged` で unstage。**修正後、承認待ちせず次ステップへ進む**
     - 以下は CRITICAL に上げない（修正不要）: プレースホルダ文字列（`your-api-key-here`, `xxxxx`, `REDACTED`, `example`, `dummy`）、`fixtures/` / `*.test.*` / `*.spec.*` 配下のダミー値。実値が見えた時点で CRITICAL 確定
   - **WARNING / INFO**（疑わしいが誤検知の可能性がある / ベストプラクティス違反）: サマリーに列挙するのみ、修正はしない
4. WARNING が 1 件以上出た場合、サマリー末尾に「より深いセキュリティレビューは `/security-review` を手動実行してください」と一文添える

### Step 3: CLAUDE.md 同期

**条件**: Step 0 で取得した `git diff --stat` の出力に `CLAUDE.md`（大小文字無視）が含まれるかを確認する。
含まれる場合のみ `claude-md-management:claude-md-improver` スキルを Skill ツールで呼び出す。
含まれない場合はスキップして次のステップへ。

### 完了: サマリー表示

全ステップの結果をまとめて表示する：

- 各ステップで発見された問題と修正内容
- 未修正の警告事項（あれば）
- ワーキングツリーの状態（`git status --short`）

## エラーハンドリング

| 状況 | 対応 |
|------|------|
| main ブランチ上 | エラー終了、ブランチ作成を促す |
| main との差分なし | 警告表示、変更がないことを報告 |

## Rules

- 各ステップは必ず順番に実行する（並列実行しない）
- 問題を発見したら自動修正し、修正内容をユーザーに報告する
- **対話的な built-in コマンド（`/security-review` 等）を自動で呼ばない** — 指摘はインラインで扱い、承認待ちで止めない
- コミット・プッシュ・PR 作成は絶対に行わない
- 修正はワーキングツリーに残し、ユーザーが確認できるようにする
