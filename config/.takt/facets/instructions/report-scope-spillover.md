これまでの run で蓄積されたレポートを読み、現 issue のスコープ外で発見された改善点を別 issue として起票してください。

**スコープ判定の絶対基準**: 「この修正を入れたら本 issue の PR タイトルが変わるか?」 変わるならスコープ外。

**現 worktree 内では新規ファイルを作成・変更しない。** 起票のみが本 step の責務。

## やること

1. 直前までの run レポートを読む（{report:plan.md} / {report:coder-scope.md} / {report:coder-decisions.md} / {report:architect-review.md} / {report:supervisor-validation.md} / {report:ai-review.md} など、存在するもの全て）
2. スコープ外候補を抽出する。対象例:
   - 無関係なテストの flakiness
   - 触ったファイルの古いコメント・型注釈不足
   - 依存ライブラリの軽微な脆弱性
   - 設計上の重複・抽象化したくなる箇所
   - 既存のリファクタ機会
3. 各候補を「PR タイトルが変わるか?」で振るう。変わるならスコープ外として残す
4. 残った候補を `gh issue list --search "<keyword>"` で重複確認したのち、`gh issue create --title "..." --body "..."` で起票する
5. 起票結果（番号・URL・タイトル）を控える

## 必須出力

### ## 検出したスコープ外項目

| # | 概要 | 出典レポート | PR タイトル変化 |
|---|------|-------------|----------------|

### ## 起票した issue

| # | issue 番号 | URL | タイトル |
|---|-----------|-----|---------|

### ## 起票しなかった項目

| # | 概要 | 起票しなかった理由（重複 / スコープ内 / 情報不足） |
|---|------|---------------------------------------------------|

スコープ外発見が 0 件なら 3 セクションとも「該当なし」と明記する。
