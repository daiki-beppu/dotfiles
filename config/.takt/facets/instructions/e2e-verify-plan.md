# E2E 動作確認 — 計画

タスク説明で指定された機能・画面について、ブラウザで動作確認するための計画を作成する。

## 手順

### 1. コードベース分析

以下を特定する:

- **dev server 起動コマンド**: `package.json` の `scripts`（`dev`, `start`, `serve` 等）を読む
- **ベース URL**: ポート番号、ホスト名（`localhost:3000` 等）
- **対象ルート/ページ**: ルーティング定義（Next.js の `app/`, React Router, Vue Router 等）からタスクに関連するルートを抽出
- **主要コンポーネント**: フォーム、ボタン、モーダル、トースト等のインタラクティブ要素

### 2. シナリオ設計

タスク説明に基づき、以下の 2 カテゴリでシナリオを設計する:

#### Happy パス
- ユーザーが正常に操作を完了できるフロー
- 各ステップで期待される画面状態を明記
- 入力値は具体的に（「テスト太郎」「test@example.com」等）

#### エラーケース
- バリデーションエラー（必須項目未入力、形式不正、境界値）
- 存在しないリソースへのアクセス（404）
- 権限エラー（未認証状態でのアクセス）
- 各ケースで期待されるエラー表示を明記

### 3. 計画の品質基準

- シナリオは最小 2 件（Happy 1 + Error 1）、最大 10 件程度
- 各シナリオに一意の ID（H-1, E-1 等）を付与
- 操作手順は「ページ遷移 → 要素操作 → 結果確認」の粒度で記述
- dev server の起動コマンドが特定できない場合は ABORT
